%% 1. Agent Installation

Please make sure that the IIS 6 Server is installed and running. Also
make sure that the IIS 6 Agent is compiled with steps indicated in README.txt.

Installation does the following:

    - Creates the IIS 6 Agent Instance Directory
    - Tag Swapping in AMAgent.properties file
    - Updating IIS 6 web site: adds agent filter

%% 1.1 Steps

1.1.1 Extract  iis_v6_WINNT_agent.zip installation bits.
1.1.2 Goto web_agents\iis6_agent\bin
1.1.3 Execute: cscript.exe IIS6CreateConfig.vbs <config_file>
Follow the installation interactions and provide these details:

    - IIS 6 Web site id
    - Agent Host name
    - Agent Host port
    - Agent Host protocol
    - Access Manager Services Host
    - Access Manager Services port
    - Access Manager Services Protocol
    - Access Manager Services Deployment URI
    - Agent Profile's plain text password (this is the password of amldapuser by default).

1.1.4 Execute: cscript.exe IIS6admin.vbs -config <config_file>

1.1.5 Restart the server

%% 3. Overriding IIS6's BASIC_AUTH mechanism. (Optional)

If BASIC_AUTH is enabled in IIS 6, IIS prompts a pop-up window to enter credentials.
This can be overridden using agent functionality. This means the additional pop-up window
won't show up. 

In order to achieve this, agent provides an additional component, authentication extension.

3.1 Adding the authentication ISAPI extension

Example:

C:\web_agents\iis6_agent\bin>cscript.exe IIS6AuthAdmin.vbs -a ADDFILTER -i
 87257621 -p c:\web_agents\iis6_agent\bin\amiis6auth.dll


3.2 Removing the authentication ISAPI extension

Example:

C:\web_agents\iis6_agent\bin>cscript.exe IIS6AuthAdmin.vbs -a REMOVEFILTER
 -i 87257621 -p c:\web_agents\iis6_agent\bin\amiis6auth.dll
